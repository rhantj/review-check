01 · 프로젝트 개요

추진 배경 및 목적

  • 게임 구매 결정에 리뷰가 큰 영향을 주지만, 인기 게임은 리뷰가 수만 건이라 전체 여론을 파악하기 어렵다.
  • 직접 학습한 딥러닝 감성분류 모델로 리뷰의 긍/부정을 판별하고, LLM으로 장단점을 자연어 요약하며, RAG로 리뷰 근거 기반 질의응답을 제공한다.
  • 동일 데이터에서 비사전학습(LSTM) vs 사전학습(DistilBERT) 모델을 비교해 전이학습의 효과를 정량적으로 검증한다.

핵심 목표

리뷰 입력 → DistilBERT 긍/부정 분류 → 긍정/부정 그룹별 LLM 장단점 요약, 그리고 질문 입력 → 벡터 검색 top-k 리뷰 근거 → LLM 답변을 제공하는 Streamlit 2탭 데모 앱 구현 및 HF Spaces 배포

감성별 판별 토큰 — 로그오즈비 분석

왜 빈도가 아닌 로그오즈비인가 · 단순 빈도 상위 토큰은 "game", "play" 같은 도메인 최빈어가 양쪽에 똑같이 나와 정보가 없다. 로그오즈비는 한쪽 감성에서 상대적으로 훨씬 자주 쓰이는 단어를 뽑아 실제 감성 신호(refund, garbage / superb, 10/10)를 드러낸다.

분석 과정에서 잡은 통계 왜곡

  • 밈 리뷰 1건이 "salad"를 48회 반복 → 부정 판별 토큰 오염 → 문서 빈도(리뷰당 1회)로 집계 전환
  • ( ͡° ͜ʖ ͡°) 밈 조각·그린텍스트 기호(>) → 영숫자 없는 토큰 제외 필터
  • 특정 게임 캐릭터명(고유명사) 혼입 → 샘플링 편향으로 명시, 해석 시 유의

03 · 모델 학습 및 비교

모델 설계 — LSTM 베이스라인 vs DistilBERT

LSTM (비사전학습 베이스라인)

공백 토크나이저→ Embedding 100d→ LSTM 128→ Linear 2

어휘·임베딩을 처음부터 학습. min_freq=2 어휘 구축, PAD=0/UNK=1. 사전지식 없이 이 데이터만으로 학습하는 대조군.

DistilBERT (사전학습 파인튜닝)

WordPiece 토크나이저→ distilbert-base-uncased→ 분류 헤드 2

대규모 코퍼스 사전학습 지식 위에 파인튜닝. HF Trainer + DataCollatorWithPadding, max_len 128.

Apple Silicon(MPS) 메모리 제약 대응

16GB 통합메모리에서 batch 16 학습 시 커널 사망(silent OOM) → batch 8 + gradient accumulation 2(유효 배치 16 유지) + max_len 256→128로 해결. 동일 조건 재현을 위해 seed 42 고정.

* DistilBERT는 균형 1만 건 재학습 예정 — 균형 test는 난이도가 높아져 수치는 내려가지만 비교는 공정해진다. 비교 구도는 고전 ML → 비사전학습 DL → 사전학습 DL 3단.

핵심 관찰 ① — 다수 클래스 붕괴

불균형(82:18) 초기 실험에서 LSTM은 부정 리뷰 447건 중 6건만 부정으로 판별, acc 0.819는 전부 긍정으로 찍은 기저율이었다. 이것이 50:50 균형 샘플링 도입의 직접 계기가 됐다.

핵심 관찰 ② — 패딩 버그 발견·해결

균형 전환 후에도 loss가 0.693(무작위 수준)에 고정 → 원인은 패딩 토큰이 LSTM hidden state를 씻어내는 구조 결함. pack_padded_sequence로 실제 길이까지만 처리하도록 수정해 acc 0.51 → 0.76. 균형 샘플링이 숨어 있던 버그를 노출한 사례.

03 · 모델 학습 및 비교

지표 비교 요약

  • 비교군을 3단으로 확장: TF-IDF+로지스틱 회귀(고전 ML) → LSTM(비사전학습 DL) → DistilBERT(사전학습 DL). 이 데이터 규모에선 고전 ML이 LSTM과 대등하거나 우세한 경우도 많아, "딥러닝 ≠ 무조건 우월, 관건은 사전학습"이라는 메시지를 정량적으로 보여준다.
  • LSTM은 val loss 기준 early stopping(최대 100에폭, patience 10)으로 최적 시점(21에폭)에서 종료.
  • 비교표는 models/comparison.md로 자동 생성되어 재학습 시 즉시 갱신.

05 · LLM 요약 · RAG Q&A

LLM 장단점 요약 및 RAG 질의응답

리뷰 입력 → DistilBERT 분류 → 긍정/부정 그룹 → Qwen3-30B-A3B-Instruct → 장단점 3개씩 요약

RAG 질의응답

train 리뷰 2,000건을 all-MiniLM-L6-v2로 임베딩해 Chroma에 색인. 질문 → 벡터 유사도 top-5 리뷰 검색 → 근거로 제시하며 LLM이 답변. 답변 화면에서 근거 리뷰 원문을 함께 노출해 환각을 억제.

LLM 추상화 설계

LLM 호출을 src/llm/client.py 한 곳으로 추상화 — 환경변수 하나로 HF Inference API(배포) ↔ Ollama(로컬) 전환. 1차 모델 실패 시 폴백 모델(gemma-2-9b-it) 자동 전환으로 데모 안정성 확보.

모델 선택 · Qwen2.5-7B → Qwen3-30B-A3B-Instruct-2507로 전환. MoE 구조로 추론이 빠르고 non-thinking 전용이라 요약 출력에 사고 과정이 섞이지 않으며, 한국어 품질이 개선됐다. 추론은 HF 서버에서 수행되어 로컬 자원과 무관.

Streamlit 2탭 · 탭1 감성분석+요약, 탭2 RAG Q&A. RAG 모듈은 지연 import로 초기 로딩 최소화.
