# 설계 문서: Steam 리뷰 감성분석 + AI 요약 + RAG Q&A

- 작성일: 2026-07-02
- 발표일(마감): 2026-07-07 (화)
- 목적: 포트폴리오/취업용 — 완성도 + 배포 데모 + 발표 스토리

## 1. 한 줄 요약

게임 리뷰를 입력하면 **직접 학습한 DL 모델**이 긍/부정을 분류하고, **LLM(Qwen2.5-Instruct)** 이 장점·단점을 자연어로 요약하며, **RAG**로 리뷰 기반 질의응답을 제공하는 웹 데모.

## 2. 데이터셋

- **Steam Reviews (Kaggle)**: `review_text` + `review_score`(추천/비추천) 라벨이 이미 붙어있어 전처리 부담이 적음.
- 발표용으로 인기 게임 1~2개 리뷰 서브셋만 샘플링(수만 건 → 수천 건)해도 충분.
- 백업: HF `datasets`의 steam 리뷰 데이터로 대체 가능.
- 원본 CSV는 `.gitignore` 처리(리포에 커밋하지 않음), 다운로드 스크립트/절차만 문서화.

## 3. 아키텍처

```
[1. 데이터 파이프라인]  raw csv → clean/split (train/val/test)
        ↓
[2. DL 감성분류]  (A) LSTM/GRU 직접학습  vs  (B) DistilBERT 파인튜닝  → 비교표
        ↓  (best model → HF Model Hub 업로드)
[3. LLM 요약]  분류된 리뷰 묶음 → Qwen2.5-Instruct (HF Inference API) → 장점/단점 요약
        ↓
[4. RAG Q&A] (Phase 2)  전체 리뷰 임베딩 → Chroma/FAISS → 질문 검색 + 근거 답변
        ↓
[5. Streamlit 앱]  탭1: 감성분석+요약 / 탭2: 리뷰 Q&A  → HF Spaces 배포
```

### 컴포넌트 경계 (각각 독립·테스트 가능)

| # | 컴포넌트 | 역할 | 입력 | 출력 | 의존 |
|---|---|---|---|---|---|
| 1 | 데이터 파이프라인 | 정제·분할 | raw csv | train/val/test | pandas |
| 2 | DL 감성분류 | 긍/부정 분류 | 리뷰 텍스트 | label + score | torch/transformers |
| 3 | LLM 요약 | 장단점 요약 | 분류된 리뷰 묶음 | 요약 텍스트 | HF Inference API |
| 4 | RAG Q&A | 검색+답변 | 질문 | 근거+답변 | 임베딩+벡터DB+LLM |
| 5 | Streamlit 앱 | UI/오케스트레이션 | 사용자 입력 | 화면 | streamlit |

## 4. 모델 결정

- **감성분류(DL)**: LSTM/GRU 직접학습 **vs** DistilBERT 파인튜닝 — 같은 데이터·같은 지표(정확도/F1/혼동행렬)로 비교. 데모 메인은 성능 좋은 쪽(예상: DistilBERT), LSTM은 비교표로 스토리 강화.
- **LLM(요약/답변)**: **Qwen2.5-Instruct**, HF Inference API로 호출. 다국어·한국어 출력에 강함.
- **Ollama**: 로컬 프롬프트 튜닝/테스트 용도만(선택). 배포본은 API 호출로 스위치. 모델 호출부를 한 곳에 추상화해 로컬↔배포 전환을 쉽게.

## 5. 앱 사용 흐름

**탭1 — 감성분석 + 요약**
1. 게임 선택(드롭다운) 또는 리뷰 텍스트 붙여넣기
2. DL 모델이 각 리뷰 긍/부정 분류 → 감성 분포 차트
3. LLM이 긍정 묶음 → "장점", 부정 묶음 → "단점" 요약
4. 결과 카드 표시

**탭2 — 리뷰 Q&A (RAG)**
1. 사용자가 자유질문 입력 ("멀미 있어?", "가격 대비 어때?")
2. 벡터DB에서 관련 리뷰 top-k 검색
3. LLM이 검색된 리뷰를 근거로 답변 + 출처 리뷰 표시

## 6. 구현 순서 (리스크 헤지)

- **Phase 1 (발표 안전판)**: 데이터 → 두 모델 학습·비교 → 요약 → Streamlit 탭1 → Spaces 배포. **여기까지가 발표 최소 완성본.**
- **Phase 2 (임팩트)**: 리뷰 임베딩 → 벡터DB → Q&A 탭2 추가. 시간 부족 시 이 탭만 제외 가능.

## 7. 스코프

- **In**: 데이터 준비, 모델 2종 비교, HF Model Hub 업로드, LLM 요약, RAG Q&A, Streamlit 2탭, HF Spaces 배포
- **Out (YAGNI)**: 실시간 Steam 크롤링, 다국어 UI, 사용자 인증, 관계형 DB

## 8. 테스트/검증

- 데이터 파이프라인: 정제·분할 단위 테스트(빈 값/중복/분할 비율)
- 모델: 평가 지표(정확도/F1/혼동행렬), 두 모델 비교표
- 앱: 스모크 테스트(입력→출력 정상 반환)

## 9. 발표 스토리 (차별점)

1. **모델 비교**: LSTM vs DistilBERT — "왜 트랜스포머가 이기는가"
2. **DL + LLM 결합**: 단순 분류를 넘어 자연어 인사이트
3. **RAG**: 자유질문에 리뷰 근거로 답변 (검색 역량)
4. **엔드투엔드 배포**: 학습 → Model Hub → Spaces
